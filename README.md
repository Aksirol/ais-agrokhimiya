# 🌱 АІС «Агрохімія» (Agrochemistry Management System)

Автоматизована інформаційна система для управління агрономічними процесами, обліку агрохімікатів (добрив, гербіцидів, фунгіцидів) та аналітики витрат. 

Цей проєкт розроблено як повноцінну ERP-систему з мікросервісним підходом, що включає суворе розмежування прав доступу (RBAC), транзакційну цілісність бази даних та контейнеризацію за допомогою Docker.

## ✨ Основний функціонал

- 🔐 **Role-Based Access Control (RBAC):** Три рівні доступу (Адміністратор, Агроном, Оператор складу) з індивідуальними можливостями та інтерфейсами.
- 📦 **Управління складом:** Відстеження залишків у реальному часі, автоматичний розрахунок статусів (Норма/Мало/Критично).
- 🛒 **Закупівлі:** Повний цикл замовлення (від заявки до прийому на склад з автоматичним поповненням залишків).
- 🚜 **Журнал використання:** Списання препаратів зі складу на конкретні поля. Логіка часткового повернення залишків при завершенні робіт.
- 📊 **Дашборд та Аналітика:** Математично генеровані SVG-діаграми (без важких сторонніх бібліотек), розумні сповіщення про низькі запаси.
- ⚙️ **Довідники:** Управління полями, хімікатами, складами та постачальниками.

## 🛠 Стек технологій

**Frontend:**
- React 18 (Vite)

## 🚀 Швидкий запуск (Docker)

Найпростіший спосіб запустити проєкт — використовувати Docker. Це автоматично розгорне базу даних, бекенд та фронтенд в єдиній мережі.

### 1. Вимоги
- Встановлений [Docker](https://www.docker.com/) та Docker Compose.

### 2. Запуск контейнерів
Клонуйте репозиторій та виконайте в кореневій папці проєкту:
```bash
docker-compose up -d --build
```

### 3. Наповнення бази даних (Сідінг)
Оскільки база даних порожня, потрібно створити тестових користувачів та базові довідники. Виконайте команду для запуску скрипта всередині бекенд-контейнера:
```bash
docker exec -it agro_backend npx prisma db seed
```

### 4. Використання
Відкрийте браузер і перейдіть за адресою: **`http://localhost`**

**Тестові акаунти:**
- Адміністратор: `ivan@agro.com` | Пароль: `admin123`
- Агроном: `petro@agro.com` | Пароль: `admin123`
- Оператор складу: `olena@agro.com` | Пароль: `admin123`

Щоб зупинити систему:
```bash
docker-compose down
```

## 💻 Локальна розробка (Без Docker)

Якщо ви хочете запустити проєкт для розробки локально:

### 1. Налаштування Бази Даних
Переконайтеся, що у вас встановлено PostgreSQL.
У папці `backend` скопіюйте файл `.env.example` у `.env` та вкажіть ваш `DATABASE_URL` та `JWT_SECRET`:
```bash
cp .env.example .env
```

### 2. Запуск Backend
```bash
cd backend
npm install
npx prisma migrate dev  # Застосування структури БД
npx prisma db seed      # Наповнення тестовими даними
npm run dev             # Запуск сервера на порту 5000
```

### 3. Запуск Frontend
Відкрийте новий термінал:
```bash
cd frontend
npm install
npm run dev             # Запуск React (Vite) на порту 5173
```

## 🏗 Особливості архітектури
- **Транзакційність (ACID):** Списання товарів на поля та повернення залишків при скасуванні заявок відбувається через `prisma.$transaction`. Це гарантує, що у разі збою товар не зникне "в нікуди".
- **Real-time пошук та сортування:** Всі таблиці оснащені клієнтським пошуком (без зайвих запитів на сервер) та інтерактивним сортуванням по колонках.
- **Розумна навігація (Command Palette):** Глобальний пошук у шапці пропонує швидкий перехід до потрібних розділів системи.
